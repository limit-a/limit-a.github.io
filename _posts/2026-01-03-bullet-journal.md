## 불렛저널(Bullet Journal) 완벽 가이드

### 들어가며

우리는 많은 일들의 행적을 보관한다.
- 우리가 한 일들
- 우리가 해야할 일들  
- 우리가 하고 싶은 것들

행적을 보관할 일들이 많다. 그걸 위한 많은 어플이 있지만, 무엇을 쏟아 놓던지 정리할 수 있을 정도로 **유연하면서도** 방해되지 않을 정도로 **충분히 빠른** 시스템이 필요했다.

불렛저널은 **과거의 행적을 보관하고, 현재를 정리하고, 미래를 계획하기 위해** 디자인된 아날로그 시스템이다.

```
┌─────────────────────────────────────┐
│   불렛저널의 3가지 핵심 목적        │
├─────────────────────────────────────┤
│  📜 과거의 행적 보관 (Track)        │
│  📋 현재를 정리 (Order)             │
│  📅 미래를 계획 (Design)            │
└─────────────────────────────────────┘
```

📝 주문제작된 노트가 아니어도 상관없다. **어떤 노트를 선택하든 똑같이 작동한다.**

---

## 📖 Index (색인)

첫 번째 빈 양면을 펼친다. 이것이 **색인**이 된다.

**설정 방법:**
- 양면 상단에 "Index" 제목 작성

이후 모든 항목의 페이지 번호를 여기에 기록하게 된다.

### Index 레이아웃

```
┌─────────────────────┬─────────────────────┐
│      INDEX          │      INDEX          │
├─────────────────────┼─────────────────────┤
│ Future Log ... 2-3  │ June 2024 .... 18-25│
│ January 2024 .. 4-9 │ Project X ..... 26-28│
│ February 2024  10-15│ Shopping List . 29   │
│ March 2024 ... 16-17│ Book List ..... 30-32│
│                     │                      │
│                     │                      │
└─────────────────────┴─────────────────────┘
    페이지 1               페이지 2
```

---

## 📅 Future Log (미래일지)

다음 빈 양면으로 넘긴다.

**설정 방법:**
1. 양면을 "Future Log"로 제목 작성
2. 페이지의 총 줄 수를 센다
3. 그 수를 3으로 나눈다
4. 자를 사용해 양면에 선을 긋는다
5. 각 박스에 월(月)을 적는다
6. 페이지 번호를 쓰고 Index에 Future Log를 기록한다

설정 방법은 여러 가지가 있지만, **이것은 간단한 6개월 버전**이다.

### Future Log 레이아웃 (6개월)

```
┌─────────────────────┬─────────────────────┐
│   FUTURE LOG        │   FUTURE LOG        │
├─────────────────────┼─────────────────────┤
│  JANUARY            │  APRIL              │
│  ○ 15일 정기검진    │  • 세금 신고        │
│  • 프로젝트 시작    │                     │
│                     │                     │
├─────────────────────┼─────────────────────┤
│  FEBRUARY           │  MAY                │
│  ○ 생일 파티        │  ○ 여행 계획        │
│                     │  • 보고서 제출      │
│                     │                     │
├─────────────────────┼─────────────────────┤
│  MARCH              │  JUNE               │
│  • 분기 보고서      │                     │
│  ○ 컨퍼런스 참석    │                     │
│                     │                     │
└─────────────────────┴─────────────────────┘
    페이지 2               페이지 3
```

## 📆 Monthly Log (매달 일지)

다음 빈 양면으로 넘긴다. 이것이 **매달 일지**가 된다.

양 페이지에 그달의 이름을 적는다.

### 왼쪽 페이지 - Monthly Calendar

왼쪽은 **달력**이 된다.

1. 그달의 모든 날짜를 적어 내려간다
2. 그날들의 요일 첫 글자를 적는다

이것이 달력이다.

### 오른쪽 페이지 - Monthly Task List

오른쪽 페이지는 **일 목록**이 된다.

1. 이번 달에 할 필요가 있는 것들을 모두 적는다
2. 각 일 앞에 간단한 점(**•**)인 task bullet을 그린다
3. 페이지 숫자를 적고 Index에 해당 월을 기록한다

**Monthly Log는 해야하는 모든 것들을 한 눈에 보여준다.**  
해야할 달과 시간을.

### Monthly Log 레이아웃

```
┌─────────────────────┬─────────────────────┐
│   JANUARY 2024      │   JANUARY 2024      │
├─────────────────────┼─────────────────────┤
│ 1  M                │ • 보고서 작성       │
│ 2  T  ○ 팀 미팅     │ • 운동 루틴 시작    │
│ 3  W                │ • 책 3권 읽기       │
│ 4  T                │ • 프로젝트 기획안   │
│ 5  F  ○ 친구 저녁   │ • 차량 정비         │
│ 6  S                │ • 집 청소           │
│ 7  S                │                     │
│ 8  M                │                     │
│ 9  T                │                     │
│ 10 W                │                     │
│ ...                 │                     │
│ 31 W                │                     │
└─────────────────────┴─────────────────────┘
  Monthly Calendar       Monthly Tasks
    (날짜+요일)           (할 일 목록)
```

## 📝 Daily Log (일지)

이제 일지를 설정한다.

**날짜를 적는 것으로 시작한다.**

일은 작고 불렛이 찍힌 문장들로 채워진다.

### Rapid Log (빠른 기록)

**각 일은 세 가지 중 하나로 분류된다:**

| 기호 | 분류 | 의미 | 예시 |
|:----:|------|------|------|
| **•** | Task | 할 일 | • 보고서 작성하기 |
| **○** | Event | 이벤트/약속 | ○ 오후 3시 미팅 |
| **-** | Note | 메모/아이디어 | - 좋은 아이디어 메모 |

### Daily Log 예시

```
┌──────────────────────────────────────────┐
│  MONDAY, JANUARY 8                       │
├──────────────────────────────────────────┤
│  • 아침 회의 준비                        │
│  ○ 10:00 팀 미팅                         │
│  - 새로운 아이디어: AI 활용 방안        │
│  * 긴급: 고객 이메일 답장                │
│  • 보고서 초안 작성                      │
│  - 점심: 김과장님과 프로젝트 논의        │
│  ○ 15:00 클라이언트 미팅                 │
│  • 운동 30분                             │
│                                          │
│  TUESDAY, JANUARY 9                      │
├──────────────────────────────────────────┤
│  * 보고서 최종 제출 (마감!)              │
│  • 데이터 분석                           │
│  - 좋은 레스토랑 발견: 맛나 식당         │
│  ○ 저녁 7시 동창 모임                    │
└──────────────────────────────────────────┘
```

### Signifier (기호 표현)

**일이 정말로 중요하다면** 점(•) 대신 **별(\*)을 쓴다.**

이것을 **기호 표현**이라 한다. 기호 표현은 다른 특별한 의미를 불렛에 더할 수 있다. 이 경우엔 우선 사항이다.

#### 기호 표현 예시

```
┌────────────────────────────────────┐
│  일반 Task:     • 이메일 확인     │
│  중요 Task:     * 계약서 검토     │
│                 (점 대신 별)       │
└────────────────────────────────────┘
```

### 💨 Rapid Logging의 의미

이것이 **빠르게 일을 적는 방법**, 즉 Rapid Logging이다.

**정보를 담고 정리하는 것을 정말로 빠르게 만든다.**

이제 불렛저널의 중요한 모든 부분을 설정했다:

```
┌───────────────────────────────────────┐
│  불렛저널 4대 핵심 요소               │
├───────────────────────────────────────┤
│  ✓ Index       - 모든 것의 목차      │
│  ✓ Future Log  - 먼 미래 계획        │

└───────────────────────────────────────┘
```

이제 이것들이 어떻게 작동하는지 알아보자.

## 🔄 Migration (이동) - 핵심 프로세스

### 매달 말 진행 과정

각 달의 말에 다음 Monthly Log를 시작한다.

### 1단계: 검토

Daily Log를 훑어보며 남은 일을 찾는다.

끝낸 일에는 **점(•)을 X로 바꿔 쓴다.**

```
┌─────────────────────────────────────┐
│  완료 표시 - 점을 X로 교체          │
├─────────────────────────────────────┤
│  처음:  • 보고서 작성               │
│          ↓ (완료 후)                │
│  결과:  X 보고서 작성               │
│         (• → X로 교체)              │
└─────────────────────────────────────┘

실제 노트 예시:
  X 보고서 작성 완료
  X 이메일 답장 완료
  • 프로젝트 기획 (미완료)
  • 운동하기 (미완료)
```

### 2단계: 자문

시간을 가지면서 남은 일들을 검토한다.

스스로에게 질문한다:

> **"이 일이 아직도 내 시간을 들일 만한 가치가 있는가?"**

### 3단계: 결정과 표시

#### 가치가 없다면 - 취소선 긋기

**전체 내용에 취소선을 그어서 취소 표시**

```
┌─────────────────────────────────────┐
│  취소 표시 - 취소선 긋기            │
├─────────────────────────────────────┤
│  처음:  • 헬스장 등록               │
│          ↓ (불필요 판단)            │
│  결과:  ~~• 헬스장 등록~~           │
│         (전체에 취소선)             │
└─────────────────────────────────────┘

실제 노트 예시:
  ~~• 헬스장 등록~~
  (더 이상 필요 없음)
```

#### 다음 달로 이동

**점(•)을 > 기호로 바꿔 쓴다**  
그리고 새로운 Monthly Log에 옮겨 적는다.

```
┌─────────────────────────────────────┐
│  다음 달 이동 - 점을 >로 교체       │
├─────────────────────────────────────┤
│  처음:  • 프로젝트 기획안           │
│          ↓ (다음 달로)              │
│  결과:  > 프로젝트 기획안           │
│         (• → > 로 교체)             │
└─────────────────────────────────────┘

실제 노트 예시:
  > 프로젝트 기획안
    (2월 Monthly Log에 • 프로젝트 기획안 으로 다시 작성)
```

#### 미래로 이동

**점(•)을 < 기호로 바꿔 쓴다**  
그리고 Future Log에 옮겨 적는다.

```
┌─────────────────────────────────────┐
│  미래 이동 - 점을 <로 교체          │
├─────────────────────────────────────┤
│  처음:  • 자격증 시험 준비          │
│          ↓ (3월로)                  │
│  결과:  < 자격증 시험 준비          │
│         (• → < 로 교체)             │
└─────────────────────────────────────┘

실제 노트 예시:
  < 자격증 시험 준비
    (Future Log 3월란에 • 자격증 시험 준비 로 작성)
```

이 과정을 **Migration(이동)**이라 한다.

### Migration 기호 정리

| 상태 | 기호 변화 | 의미 | 다음 단계 |
|------|-----------|------|-----------|
| 완료 | **•** → **X** | 점을 X로 교체 | 없음 |
| 취소 | **• 내용** → **~~• 내용~~** | 전체에 취소선 | 없음 |
| 다음 달 | **•** → **>** | 점을 >로 교체 | 새 Monthly Log에 기록 |
| 미래 | **•** → **<** | 점을 <로 교체 | Future Log에 기록 |

### Migration 실제 예시

```
┌──────────────────────────────────────────────┐
│  1월 Daily Log (월말 검토)                    │
├──────────────────────────────────────────────┤
│  X 보고서 제출                                │
│    (• → X로 교체, 완료)                       │
│                                               │
│  ~~• 헬스장 등록~~                            │
│    (취소선, 불필요)                           │
│                                               │
│  > 프로젝트 기획안                            │
│    (• → >로 교체, 2월로 이동)                │
│                                               │
│  < 자격증 시험 준비                           │
│    (• → <로 교체, 3월로 이동)                │
└──────────────────────────────────────────────┘
              ↓
┌──────────────────────────────────────────────┐
│  2월 Monthly Log (새로 작성)                  │
├──────────────────────────────────────────────┤
│  • 프로젝트 기획안 (1월에서 이동)             │
│  • 2월 신규 업무들...                         │
└──────────────────────────────────────────────┘
              ↓
┌──────────────────────────────────────────────┐
│  Future Log - March                           │
├──────────────────────────────────────────────┤
│  • 자격증 시험 준비 (1월에서 이동)            │
└──────────────────────────────────────────────┘
```

### Migration 결정 흐름도

```
                미완료된 일 발견
                   (• 표시)
                      ↓
        ┌─────────────────────────┐
        │ 시간 들일 가치 있는가?  │
        └─────────────────────────┘
                      ↓
         ┌────────────┴────────────┐
         │                         │
      [NO]                      [YES]
         │                         │
         ↓                         ↓
   ~~• 내용~~             ┌───────────────────┐
   (취소선 긋기)          │ 언제 할 일인가?   │
                          └───────────────────┘
                                   ↓
                      ┌────────────┴────────────┐
                      │                         │
                  [다음 달]                [몇 달 후]
                      │                         │
                      ↓                         ↓
                 • → >                     • → <
                 점을 >로                  점을 <로
                 교체                      교체
                      ↓                         ↓
                 다음 Monthly           Future Log에
                 Log에 •로              •로 새로
                 새로 작성              작성
```

### 기호 교체 과정 시각화

```
┌───────────────────────────────────────────────┐
│  불렛저널 기호 변화 (펜으로 교체)             │
└───────────────────────────────────────────────┘

1. 처음 작성할 때:
   • 보고서 작성
   • 운동하기
   • 미팅 참석
   • 헬스장 등록

2. 나중에 상태 변경:
   X 보고서 작성             • → X
   > 운동하기                • → >
   ~~• 미팅 참석~~           • → 취소선
   < 헬스장 등록             • → <
```

## 🎯 Migration의 목적

**Migration은 주의를 산만하게 만드는 것을 지워낼 수 있도록 돕는다.**

**시간을 들일 만한 가치가 있는 일들에 집중하도록 설계되었다.**

### 핵심 철학

```
┌────────────────────────────────────────┐
│                                        │
│    바쁘게 있는 것 ≠ 생산적이게 있는 것 │
│                                        │
│    Migration은 이 차이를 만든다        │
│                                        │
└────────────────────────────────────────┘
```

## 📚 Collection (수집물)

가끔 특정 주제와 관련된 일과 메모가 생긴다.

관련있는 것들을 정리하기 위해 **Collection**을 만든다.

### 만드는 방법

1. 빈 페이지로 간다
2. 주제를 적고 페이지 번호를 매긴다
3. 모든 메모나 일을 Collection에 기록한다
4. 나중에 참고하기 위해 Index에 Collection을 기록한다

**Collection은 쇼핑 목록, 진행 중인 계획, 수업 노트 등을 정리하는 훌륭한 방법이다.**

### Collection 예시들

#### 예시 1: 쇼핑 목록

```
┌──────────────────────────────────────┐
│  SHOPPING LIST              p.26     │
├──────────────────────────────────────┤
│  • 우유                              │
│  • 계란                              │
│  • 빵                                │
│  • 과일 (사과, 바나나)               │
│  • 세제                              │
│  • 휴지                              │
│                                      │
│  (구매 완료시 점을 X로 교체)         │
└──────────────────────────────────────┘
```

#### 예시 2: 프로젝트 관리

```
┌──────────────────────────────────────┐
│  PROJECT X                  p.27-29  │
├──────────────────────────────────────┤
│  ○ 1/15 킥오프 미팅                  │
│  • 요구사항 문서 작성                │
│  • 팀원 역할 분담                    │
│  - 참고: 작년 프로젝트 참조          │
│  • 1차 프로토타입 제작               │
│  ○ 2/1 중간 검토                     │
│  • 최종 발표 준비                    │
└──────────────────────────────────────┘
```

#### 예시 3: 읽고 싶은 책

```
┌──────────────────────────────────────┐
│  BOOKS TO READ              p.30-32  │
├──────────────────────────────────────┤
│  • 총, 균, 쇠 - 재레드 다이아몬드    │
│  X 사피엔스 - 유발 하라리            │
│    (읽음 - 점을 X로 교체)            │
│  • 코스모스 - 칼 세이건              │
│  • 1984 - 조지 오웰                  │
```

## 🖊️ 불렛저널의 핵심: 기호 교체 시스템

불렛저널은 **펜으로 종이에 작성**하는 아날로그 시스템이다.  
디지털처럼 삭제하고 다시 쓰는 것이 아니라, **처음 쓴 점(•)을 다른 기호로 바꿔 쓰면서 상태를 변경**한다.

### 기호 교체 규칙

```
┌──────────────────────────────────────────────┐
│  기본 원칙: 점(•)을 다른 기호로 교체         │
├──────────────────────────────────────────────┤
│                                              │
│  • → X           (완료: 점을 X로)           │
│  • → >           (다음 달: 점을 >로)        │
│  • → <           (미래: 점을 <로)           │
│  • → *           (중요: 점을 *로)           │
│  • 내용 → ~~• 내용~~  (취소: 취소선)        │
│                                              │
└──────────────────────────────────────────────┘
```

### 실제 노트 작성 흐름

```
월요일 아침 (처음 작성):
  • 보고서 작성
  • 운동하기
  • 이메일 답장
  • 미팅 준비

월요일 저녁 (상태 업데이트 - 기호 교체):
  X 보고서 작성             완료! • → X
  > 운동하기                내일로! • → >
  X 이메일 답장             완료! • → X
  ~~• 미팅 준비~~           취소! • → 취소선

화요일 (새로 작성):
  • 운동하기 (어제서 이동)
  • 새로운 할 일들...
```

```
│                                             │
```

## 🎯 핵심 포인트 요약

| 요소 | 목적 | 빈도 | 특징 |
|------|------|------|------|
| **Index** | 전체 목차 | 필요시 업데이트 | 찾기 쉽게 |
| **Future Log** | 장기 계획 | 초기 설정 후 필요시 | 6개월 개요 |
| **Monthly Log** | 월간 개요 | 매달 초 | 달력+할일 |
| **Daily Log** | 일일 기록 | 매일 | Rapid Logging |
| **Migration** | 우선순위 정리 | 매달 말 | **기호 교체** |
| **Collection** | 주제별 정리 | 필요시 | 유연한 활용 |

### 주요 기호 정리

```
┌──────────────────────────────────────┐
│  기본 기호                           │
├──────────────────────────────────────┤
│  •  Task (할 일)                     │
│  ○  Event (이벤트)                   │
│  -  Note (메모)                      │
│  *  중요 표시                        │
├──────────────────────────────────────┤
│  상태 변화 기호 (점 교체)            │
├──────────────────────────────────────┤
│  X  완료 (• → X)                     │
│  >  다음 달로 이동 (• → >)           │
│  <  미래로 이동 (• → <)              │
│  ~~  취소 (취소선)                   │
└──────────────────────────────────────┘
```
